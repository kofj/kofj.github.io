Golang 用 gRPC 构建高性能 API

2017-09-27

  作为拥有多年经验的软件开发者，我们都知道 API 在系统当中的重要性——它们无处不在，是应用程序的基石，定义着应用程序之间怎么样通讯。日益增进的技术，爆发式增长的用户需求都在不断的让我们需要构建更加性能的 API。

  gRPC定义服务简便；夸语言、跨平台；快捷且可扩展；全双工通讯并内置认证。Golang 原生支持高并发；程序执行高效，语言设计优秀。
  gPRC 和 Golang 结合起来则能够让我们非常方便的开发出高性能的 API。

0x1 gRPC 与 Protocol Buffers 是什么

  RPC 是 Remote Procedure Call 的缩写，gRPC 是 Google 开源的 RPC 框架和库。gRPC 使用 Protocol Buffers 作为其序列化协议。
